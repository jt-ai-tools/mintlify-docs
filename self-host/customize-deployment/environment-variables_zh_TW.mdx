這是用於配置 Lightdash 部署的所有環境變數參考。

| 變數 | 描述 |
| :------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `PGHOST` | **(必填)** 用於儲存 Lightdash 數據的 Postgres 伺服器主機名稱 |
| `PGPORT` | **(必填)** 用於儲存 Lightdash 數據的 Postgres 伺服器連接埠 |
| `PGUSER` | **(必填)** 造訪 Postgres 伺服器以儲存 Lightdash 數據的使用者名稱 |
| `PGPASSWORD` | **(必填)** `PGUSER` 的密碼 |
| `PGDATABASE` | **(必填)** Postgres 伺服器內用於儲存 Lightdash 數據的資料庫名稱 |
| `PGCONNECTIONURI` | 用於儲存 Lightdash 數據的 Postgres 伺服器連接 URI，格式為 `postgresql://user:password@host:port/db?params`。這是提供上述個別 PG 變數的替代方案。 |
| `PGMAXCONNECTIONS` | 資料庫的最大連接數 |
| `PGMINCONNECTIONS` | 資料庫的最小連接數 |
| `LIGHTDASH_SECRET` | **(必填)** 用於保護 Lightdash 中各種權杖的秘密金鑰。在不同部署之間**必須**保持固定。若秘密金鑰變更，您將無法造訪 Lightdash 數據。 |
| `SECURE_COOKIES` | 僅允許透過 HTTPS 連接儲存 Cookie。我們使用 Cookie 保持您的登入狀態。生產環境建議設為 `true`。(預設為 `false`) |
| `COOKIES_MAX_AGE_HOURS` | 使用者對話在自動登出前持續的小時數。例如設為 24，則使用者在閒置 24 小時後會被自動登出。 |
| `TRUST_PROXY` | 告知 Lightdash 伺服器可以信任請求中的 `X-Forwarded-Proto` 標頭。若您在可信任的 HTTPS 終端代理後方使用 `SECURE_COOKIES=true`，此項非常有用。(預設為 `false`) |
| `SITE_URL` | 代管 Lightdash 的網站網址。應包含通訊協定。例如 `https://lightdash.mycompany.com` (預設為 `http://localhost:8080`) |
| `INTERNAL_LIGHTDASH_HOST` | 當您的 Lightdash 執行體無法從網際網路造訪時，供無頭瀏覽器發送請求的內部 Lightdash 主機。若 `SECURE_COOKIES=true` 則需支援 `https` (預設與 `SITE_URL` 相同) |
| `STATIC_IP` | 伺服器固定 IP，以便使用者將其加入倉庫白名單。(預設為 `http://localhost:8080`) |
| `LIGHTDASH_QUERY_MAX_LIMIT` | 查詢最大行數限制 (預設為 `5000`) |
| `LIGHTDASH_QUERY_DEFAULT_LIMIT` | 查詢預設回傳的行數 (預設為 `500`) |
| `LIGHTDASH_QUERY_MAX_PAGE_SIZE` | 分頁查詢的最大頁面大小 (預設為 `2500`) |
| `SCHEDULER_ENABLED` | 啟用/停用觸發定時交付的排程工作節點。(預設為 `true`) |
| `SCHEDULER_CONCURRENCY` | 可同時處理的定時交付作業數量。(預設為 `3`) |
| `SCHEDULER_JOB_TIMEOUT` | 作業超時時間（毫秒），超時後排程器會釋放資源供其他作業使用。(預設為 `600000`，即 10 分鐘) |
| `SCHEDULER_SCREENSHOT_TIMEOUT` | 進行截圖的超時時間（毫秒） |
| `SCHEDULER_INCLUDE_TASKS` | 要包含的排程任務清單（以逗號分隔） |
| `SCHEDULER_EXCLUDE_TASKS` | 要排除的排程任務清單（以逗號分隔） |
| `LIGHTDASH_CSV_CELLS_LIMIT` | CSV 檔案匯出的最大儲存格數 (預設為 `100000`) |
| `LIGHTDASH_CHART_VERSION_HISTORY_DAYS_LIMIT` | 配置圖表版本歷史保留的天數 (預設為 `3`) |
| `LIGHTDASH_PIVOT_TABLE_MAX_COLUMN_LIMIT` | 配置樞紐分析表的最大欄位數 (預設為 `200`) |
| `GROUPS_ENABLED` | 啟用/停用群組功能 (預設為 `false`) |
| `CUSTOM_VISUALIZATIONS_ENABLED` | 啟用/停用自定義圖表功能 (預設為 `false`) |
| `LIGHTDASH_MAX_PAYLOAD` | 最大 HTTP 請求主體大小 (預設為 `5mb`) |
| `LIGHTDASH_LICENSE_KEY` | Lightdash 企業版授權金鑰。詳情請參閱[企業版授權金鑰](/self-host/customize-deployment/enterprise-license-keys_zh_TW)。[獲取您的金鑰](https://calendly.com/lightdash-cloud/enterprise) |
| `HEADLESS_BROWSER_HOST` | 無頭瀏覽器的主機名稱 |
| `HEADLESS_BROWSER_PORT` | 無頭瀏覽器的連接埠 (預設為 `3001`) |
| `ALLOW_MULTIPLE_ORGS` | 若設為 `true`，在 Lightdash 註冊的新使用者將擁有獨立的組織，與他人分開 (預設為 `false`) |
| `LIGHTDASH_MODE` | Lightdash 模式（default, demo, pr 等）(預設為 `default`) |
| `DISABLE_PAT` | 停用個人存取權杖 (預設為 `false`) |
| `PAT_ALLOWED_ORG_ROLES` | 允許使用個人存取權杖的組織角色清單（以逗號分隔）(預設為「所有角色」) |
| `PAT_MAX_EXPIRATION_TIME_IN_DAYS` | 個人存取權杖的最大有效天數 |
| `MAX_DOWNLOADS_AS_CODE` | 儀表板即代碼的最大下載數量 (預設為 `100`) |
| `EXTENDED_USAGE_ANALYTICS` | 啟用擴展使用量分析 (預設為 `false`) |
| `USE_SECURE_BROWSER` | 為無頭瀏覽器使用安全的 WebSocket 連接 (預設為 `false`) |
| `DISABLE_DASHBOARD_COMMENTS` | 停用儀表板留言功能 (預設為 `false`) |
| `ORGANIZATION_WAREHOUSE_CREDENTIALS_ENABLED` | 啟用組織倉庫設定 (預設為 `false`) |
| `HEADWAY_ENABLED` | 在 Lightdash 選單中啟用 Headway 更新日誌組件 (預設為 `true`) |
| `SAVED_METRICS_TREE_ENABLED` | 在指標畫布視圖中啟用[儲存的樹狀圖](https://docs.lightdash.com/guides/metrics-catalog/canvas#saved-trees) (預設為 `false`) |

Lightdash 也接受所有 [標準 Postgres 環境變數](https://www.postgresql.org/docs/9.3/libpq-envars.html)

## 持久化下載 URL (Persistent download URLs)

啟用後，CSV 與儀表板 ZIP 匯出會回傳穩定的 Lightdash 代管 URL（例如 `https://lightdash.example.com/api/v1/file/{id}`），而非直接的 S3 簽名網址。每次造訪該網址時，Lightdash 會生成一個短效的 S3 簽名網址並進行導向 —— 這樣底層網址永不過期，且下載連結不會因 IAM 憑證輪換而失效。

| 變數 | 描述 |
| :--------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `PERSISTENT_DOWNLOAD_URLS_ENABLED` | 啟用持久化下載 URL (預設為 `false`) |
| `PERSISTENT_DOWNLOAD_URL_EXPIRATION_SECONDS` | 持久化 URL 的可造訪時長 (預設為 `259200`，即 3 天)。啟用此功能後，將忽略 `S3_EXPIRATION_TIME`，且每次導向生成的簽名網址會在 5 分鐘後過期。 |
| `PERSISTENT_DOWNLOAD_URL_EXPIRATION_SECONDS_EMAIL` | 覆蓋電子郵件下載連結的到期時間。若未設定則退回到上述設定。 |
| `PERSISTENT_DOWNLOAD_URL_EXPIRATION_SECONDS_SLACK` | 覆蓋 Slack 下載連結的到期時間。 |
| `PERSISTENT_DOWNLOAD_URL_EXPIRATION_SECONDS_MSTEAMS` | 覆蓋 MS Teams 下載連結的到期時間。 |

</Note>

<Info>
  **初始化執行體僅適用於 Lightdash 企業版方案。**
</Info>

| 變數 | 描述 |
| :------------------------------------ | :------------------------------------------------------------------------------------------------ |
| `LD_SETUP_ADMIN_EMAIL` | **(必填)** 初始設定的管理員電子郵件 |
| `LD_SETUP_ORGANIZATION_NAME` | **(必填)** 組織名稱 |
| `LD_SETUP_ADMIN_API_KEY` | **(必填)** 管理員 API 金鑰，必須以 `ldpat_` 開頭 |
| `LD_SETUP_PROJECT_NAME` | **(必填)** 專案名稱 |
| `LD_SETUP_GITHUB_PAT` | **(必填)** GitHub 個人存取權杖 |
| `LD_SETUP_GITHUB_REPOSITORY` | **(必填)** dbt 專案的 GitHub 儲存庫 |

  `AUTH_ENABLE_OIDC_TO_EMAIL_LINKING=true`
</Warning>
