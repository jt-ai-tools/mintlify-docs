維度通常與 dbt 模型中的欄位一一對應（例外情況請參閱 [額外維度](#額外維度) 章節）。

## 向專案新增維度

欲了解更多關於如何向專案新增維度的資訊，請參考我們的[入門指南](/get-started/develop-in-lightdash/how-to-create-dimensions_zh_TW)。

若要讓維度出現在 Lightdash 中，您只需在 dbt 模型的 YAML 檔案中宣告它即可。

```yaml
models:
  - name: my_model
    columns:
      - name: user_id # 在 Lightdash 中將顯示為 "User id"
        description: "使用者的唯一識別碼。"
```


## 維度配置

您可以在 dbt 模型 YAML 檔案的 `meta` 標籤下自定義維度。語法取決於您的 dbt 版本。

如果您想基於同一欄位宣告多個維度，請參閱 [額外維度](#額外維度) 章節。

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    models:
      - name: sales_stats
        meta:
          group_details:
            finance:
              label: 財務
              description: 財務相關欄位。
          joins:
            - join: web_sessions
              sql_on: ${web_sessions.date} = ${sales_stats.date}
        columns:
          - name: revenue_gbp_total_est
            description: '根據財務團隊預測估算的英鎊總營收。'
            meta:
              dimension:
                type: number
                label: '總營收' # 這是您在 Lightdash 中看到的標籤
                description: '我的自定義描述' # 您可以在此覆蓋 Lightdash 中顯示的描述
                sql: 'IF(${TABLE}.revenue_gbp_total_est = NULL, 0, ${registered_user_email})' # 用於定義維度的自定義 SQL
                hidden: false
                format: '[$£]#,##0.00' # 英鎊格式，捨入至小數點後兩位
                groups: ['finance']
          - name: forecast_date
            description: '預測日期。'
            meta:
              dimension:
                type: date
                time_intervals: ['DAY', 'WEEK', 'MONTH', 'QUARTER'] # 選填：日期的預設間隔為 `['DAY', 'WEEK', 'MONTH', 'YEAR']`
                urls:
                  - label: '在預測工具中開啟'
                    url: 'https://finance.com/forceasts/weeks/${ value.raw }'
                  - label: '在 Google 日曆中開啟'
                    url: 'https://calendar.google.com/calendar/u/0/r/day/${ value.formatted |split: "-" |join: "/"}'
                required_attributes:
                  is_admin: 'true'
    ```

下表列出了您可以自定義的所有維度屬性：

| 屬性 | 必填 | 數值 | 描述 |
| :------- | :------- | :----- | :---------- |
| label | 否 | string | 自定義標籤。設定此屬性後，這將是您在 Lightdash 中看到的名稱，而非維度名稱。 |
| [type](#類型-type) | 否 | 維度類型 | 維度類型會自動從 Lightdash 中的表格 schema 抓取，但您可以使用此屬性進行覆蓋。 |
| [description](#描述-description) | 否 | string | Lightdash 中維度的描述。您可以用它來覆蓋 dbt 中的描述。 |
| sql | 否 | string | 套用於定義維度之欄位的自定義 SQL。 |
| [time_intervals](#時間間隔-time-intervals) | 否 | 'default' 或 OFF 或包含 [日期](#日期選項-date-options)、[數值](#數值選項-numeric-options) 或 [字串](#字串選項-string-options) 選項的陣列[] | 'default'（或未設定時）對於日期類型會轉為 ['DAY', 'WEEK', 'MONTH', 'QUARTER', 'YEAR']，對於時戳類型則包含 'RAW'。若不想要時間間隔，請設為 'OFF'。 |
| hidden | 否 | boolean | 若設為 true，則在 Lightdash 中隱藏該維度。預設為 false。隱藏的維度也會從「查看底層數據」中排除。 |
| [compact](#簡寫-compact) | 否 | string | 此選項會簡寫數值（例如：1,500 變為 1.50K）。支援：['thousands', 'millions', 'billions', 'trillions', 'kilobytes', 'megabytes', 'gigabytes', 'terabytes', 'petabytes', 'kibibytes', 'mebibytes', 'gibibytes', 'tebibytes', 'pebibytes'] |
| [format](#格式-format) | 否 | string | 格式化結果表格與 CSV 匯出的輸出值。支援試算表風格格式（如 #,##0.00）。可參考 [customformats.com](https://customformats.com)。 |
| [groups](#群組-groups) | 否 | string 或 string[] | 設定此屬性後，維度會在側邊欄中與具有相同群組標籤的其他維度歸類在一起。 |
| [urls](#網址-urls) | 否 | `url`, `label` 的陣列 | 允許使用者點擊維度值並執行動作（如開啟外部連結）。支援 Liquid 範本來根據數值自定義連結。 |
| [required_attributes](#必要屬性-required-attributes) | 否 | 包含 `user_attribute`, `value` 的物件 | 限制具備特定屬性的使用者造訪（AND 邏輯 —— 需全部符合）。 |
| [any_attributes](#任一屬性-any-attributes) | 否 | 包含 `user_attribute`, `value` 的物件 | 限制具備特定屬性的使用者造訪（OR 邏輯 —— 至少符合一個）。 |
| [colors](#顏色-color) | 否 | 包含 `value`, `color` 的物件 | 圖表中特定數值對應的顏色。 |
| [image](#圖片顯示-image-display) | 否 | 包含 `url` 的物件 | **[開發中]** 使用 URL 範本在表格儲存格中顯示圖片。支援 LiquidJS 動態 URL。 |


## 類型 (Type)

維度的類型會自動從您的數據倉庫中抓取。您可以使用 .yml 檔案中的 `type` meta 標籤來覆蓋這些類型。如果您執行 `lightdash generate` 生成 YAML，Lightdash 會自動將數據倉庫中的類型加入。

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    - name: user_created_date
      meta:
        dimension:
          type: date
    ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
    ```yaml
    - name: user_created_date
      config:
        meta:
          dimension:
            type: date
    ```
  </Tab>
  <Tab title="Lightdash YAML">
    ```yaml
    dimensions:
      - name: user_created_date
        type: date
    ```
  </Tab>
</Tabs>

目前支援的維度類型：

| 維度類型 |
| :-------------- |
| string (字串) |
| number (數字) |
| timestamp (時戳) |
| date (日期) |
| boolean (布林) |

## 描述 (Description)

YAML 檔案中的欄位描述會自動抓取到 Lightdash 中，當您將滑鼠懸停在維度名稱上時即可看到。

<Frame>
  ![](/images/references/screenshot-column-descriptions-3498d7b3c09c38fda5b755ce21358b2c.png)
</Frame>

描述支援任何 YAML 格式，以下是三種最常用的字元：

### 引號用於轉義

當您使用雙引號或單引號包圍文字時，它會轉義其中的文字，使 YAML 識別的任何特殊字元仍能傳遞到 Lightdash UI。

```yaml
description: '此欄位的內容包含 this & that。'
```

### 大於符號用於摺疊文字塊

使用 `>-` 允許您在 YAML 檔案中編寫多行描述，但在解析時會被合併為單行。`lightdash generate` 指令會自動添加此符號以保持 YAML 易讀。

YAML 中的描述：

```yaml
- name: product_tier
  description: >-
    這是一個較長的描述...
    ...需要多行編寫
    但在 Lightdash UI 中會被合併
```

在 Lightdash UI 中顯示：

```yaml
這是一個較長的描述......需要多行編寫 但在 Lightdash UI 中會被合併
```

### 垂直線用於保留換行

如果您希望換行符號在 Lightdash UI 中維持原樣，可以使用 `|` 字元：

```yaml
- name: product_tier
  description: |
    這是一個較長的描述...
    ...需要多行編寫
    它會維持多行顯示
```

在 Lightdash UI 中顯示：

```yaml
這是一個較長的描述...
...需要多行編寫
它會維持多行顯示
```

### 使用 dbt doc blocks

您也可以在描述中使用 dbt docs blocks，[詳情請參閱此處](/guides/cli/how-to-auto-generate-schema-files_zh_TW#使用-doc-blocks-構建更好的-yml-檔案)。

## 格式 (Format)

您可以使用 `format` 參數讓維度在 Lightdash 中以特定格式顯示。Lightdash 為所有維度類型支援試算表風格的格式表達式。

要建立格式表達式，我們建議與其使用 https://customformats.com/。

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    - name: us_revenue
      meta:
        dimension:
          type: number
          description: '以美金計價的營收，保留兩位小數，簡寫為千位'
          format: '$#,##0.00," K"' # 505,430 將顯示為 '$505.43 K'
        additional_dimensions:
          percent_of_global_revenue:
            type: number
            description: '美金營收佔全球總營收的百分比。'
            sql: ${us_revenue} / ${global_revenue}
            format: '0.00%' # 0.67895243 將顯示為 '67.89%'
    ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
    ```yaml
    - name: us_revenue
      config:
        meta:
          dimension:
            type: number
            description: '以美金計價的營收，保留兩位小數，簡寫為千位'
            format: '$#,##0.00," K"' # 505,430 將顯示為 '$505.43 K'
          additional_dimensions:
            percent_of_global_revenue:
              type: number
              description: '美金營收佔全球總營收的百分比。'
              sql: ${us_revenue} / ${global_revenue}
              format: '0.00%' # 0.67895243 將顯示為 '67.89%'
    ```
  </Tab>
  <Tab title="Lightdash YAML">
    ```yaml
    dimensions:
      - name: us_revenue
        type: number
        description: '以美金計價的營收，保留兩位小數，簡寫為千位'
        format: '$#,##0.00," K"' # 505,430 將顯示為 '$505.43 K'
      - name: percent_of_global_revenue
        type: number
        description: '美金營收佔全球總營收的百分比。'
        sql: ${us_revenue} / ${global_revenue}
        format: '0.00%' # 0.67895243 將顯示為 '67.89%'
    ```
  </Tab>
</Tabs>

## 簡寫 (Compact)

您可以在 YAML 中簡寫數值。例如，如果我想將所有營收數值以千為單位顯示（如 `1,500` 顯示為 `1.50K`），我可以在 .yml 中這樣寫：

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    models:
      - name: sales
        columns:
          - name: revenue
            meta:
              dimension:
                compact: thousands # 您也可以使用 'K'
    ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
    ```yaml
    models:
      - name: sales
        columns:
          - name: revenue
            config:
              meta:
                dimension:
                  compact: thousands # 您也可以使用 'K'
    ```
  </Tab>
  <Tab title="Lightdash YAML">
    ```yaml
    type: model
    name: sales

    dimensions:
      - name: revenue
        compact: thousands # 您也可以使用 'K'
    ```
  </Tab>
</Tabs>

### 在其他維度中引用時間間隔

您可以在其他維度中引用特定維度的時間間隔。當您為一個維度（如 `session_start`）定義時間間隔時，Lightdash 會為每個間隔建立獨立的維度（如 `session_start_day`, `session_start_month`）。您可以在其他維度的自定義 SQL 中引用這些間隔。

例如，如果您有一個定義了時間間隔的 `user_created_at` 維度，您可以使用 `DAY` 間隔來計算兩個日期之間的時長：

```yaml
  - name: user_created_at
    meta:
      dimension:
        type: timestamp
        time_intervals:
        - DAY_OF_WEEK_NAME
        - WEEK
        - MONTH
        - RAW
        - DAY
        - HOUR_OF_DAY_NUM
        - QUARTER
  - name: first_purchase_at
    meta:
      dimension:
        type: timestamp
        time_intervals:
        - DAY
        - MONTH
        - QUARTER
  - name: duration
    meta:
      dimension:
        type: number
        sql: EXTRACT(DAY FROM ${first_purchase_at_day} - ${user_created_at_day})
```

在此範例中，`${user_created_at_day}` 與 `${first_purchase_at_day}` 引用了對應維度的 `DAY` 時間間隔版本。

### Liquid 範本 (Liquid Templating)

使用範本可以根據查詢結果配置 URL 數值，讓您的連結具備動態性。

**可用的 Liquid 標籤**

| 標籤 | 描述 |
| :------------------------------------------ | :------------------------------------------------------------------------------------------ |
| `${ value.formatted }` | Lightdash UI 中顯示的精確維度值。例如 `$1,427.20` |
| `${ value.raw }` | 底層 SQL 查詢回傳的原始數值。例如 `1427.2` |
| `${ row.table_name.column_name.formatted }` | Lightdash UI 中顯示的其他欄位的精確值。 |
| `${ row.table_name.column_name.raw }` | 底層 SQL 查詢回傳的其他欄位的原始數值。 |

**可用的 Liquid 過濾器**

過濾器可用於對數值進行微調轉換：

- `url_encode`：將字串編碼為 URL 安全格式，例如將空格替換為 `%20`。

  ```liquid
  ${ value.formatted | url_encode }
  ```

- `downcase`：將字串轉換為小寫。

  ```liquid
  ${ value.formatted | downcase }
  ```

- `append`：在字串後附加內容。

  ```liquid
  ${ value.formatted | append: ".html" }
  ```

更多過濾器請參考 [Liquid 官方文件](https://liquidjs.com/filters/overview.html)。


## 必要屬性 (Required attributes)

Lightdash 可以使用「使用者屬性」來限制某些維度僅供特定使用者造訪。

  <Tab title="dbt v1.10+ 與 Fusion">

      - salary:
        description: 使用者薪資

              required_attributes:
                is_admin: "true"

  </Tab>
  <Tab title="Lightdash YAML">

## 任一屬性 (Any attributes)

`required_attributes` 使用 AND 邏輯（所有條件都必須符合），而 `any_attributes` 則使用 OR 邏輯 —— 使用者只需符合**至少一個**條件即可造訪。

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    columns:
      - name: salary
        description: 使用者薪資
        meta:
          dimension:
            any_attributes:
              department: ["hr", "finance"]
              role: "manager"
    ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
    ```yaml
    columns:
      - name: salary
        description: 使用者薪資
        config:
          meta:
            dimension:
              any_attributes:
                department: ["hr", "finance"]
                role: "manager"
    ```
  </Tab>
  <Tab title="Lightdash YAML">
    ```yaml
    dimensions:
      - name: salary
        description: 使用者薪資
        any_attributes:
          department: ["hr", "finance"]
          role: "manager"
    ```
  </Tab>
</Tabs>

在此範例中，具有 `department = "hr"` 或 `department = "finance"` 或 `role = "manager"` 的使用者都可以看到 `salary` 維度。

您可以在同一個維度上結合使用 `required_attributes` 與 `any_attributes`。當兩者皆設定時，兩項檢查都必須通過。詳情請參閱 [使用者屬性](/references/workspace/user-attributes_zh_TW#合併使用-required_attributes-與-any_attributes)。

## 圖片顯示 (Image Display)

<Warning>
  **開發中功能**：此功能目前仍在開發階段，未來可能會有所變動。
</Warning>

您可以配置維度使用 URL 範本在表格儲存格中顯示圖片。圖片會以縮圖形式呈現，滑鼠懸停時會彈出工具提示顯示較大的預覽圖。這非常適合用於視覺化產品圖片、使用者頭像或其他任何基於圖片的數據。

### 顯示行為

- **縮圖**：圖片在表格儲存格中以縮圖顯示（預設高度 32px）。
- **懸停預覽**：滑鼠懸停於縮圖上時，會透過工具提示顯示較大尺寸的預覽圖。
- **錯誤處理**：圖片載入失敗時會顯示照片關閉圖示，並在工具提示中提供錯誤詳情。
- **驗證**：系統會捕捉無效的 URL 語法並顯示錯誤狀態。


### 圖片配置屬性

`image` 物件支援以下屬性：

| 屬性 | 類型 | 預設值 | 描述 |
| :------- | :----- | :------ | :---------- |
| `url` | string | (必填) | 圖片 URL。支援 LiquidJS 範本，如 `${value.raw}`, `${row.table.column.raw}` 等。 |
| `width` | number | auto | 圖片寬度（像素）。若指定寬度未指定高度，高度預設為 'auto' 以維持長寬比。 |
| `height` | number | 32 | 圖片高度（像素）。 |
| `fit` | string | "cover" | 控制圖片在維度內的填滿行為。選項：`"cover"`, `"contain"`, `"fill"`, 或 `"none"`。 |

**縮放選項 (Fit options)：**

- **`cover`** (預設) - 圖片填滿空間並維持長寬比。可能會裁切內容。
- **`contain`** - 縮放圖片以適應尺寸並維持長寬比。不會裁切。
- **`fill`** - 拉伸圖片以填滿尺寸。可能會導致圖片變形。
- **`none`** - 以原始尺寸顯示圖片，不進行縮放。

**範例：**

```yaml
# 正方形縮圖，使用 cover 填充
image:
  url: "${value.raw}"
  width: 50
  height: 50
  fit: "cover"

# 寬幅縮圖，維持長寬比
image:
  url: "https://example.com/images/${value.raw}.jpg"
  width: 100
  # height 預設為 'auto' 以維持長寬比

# 原始尺寸顯示
image:
  url: "${value.raw}"
  fit: "none"
```

### URL 範本

`image.url` 屬性支援 LiquidJS 範本，具備與[維度 URL](#網址-urls) 相同的標籤與過濾器：

**可用的 Liquid 標籤：**

| 標籤 | 描述 |
| :------------------------------------------ | :------------------------------------------------------------------------------------------ |
| `${ value.formatted }` | Lightdash UI 中顯示的精確維度值 |
| `${ value.raw }` | 底層 SQL 查詢回傳的原始數值 |
| `${ row.table_name.column_name.formatted }` | Lightdash UI 中顯示的其他欄位的精確值 |
| `${ row.table_name.column_name.raw }` | 底層 SQL 查詢回傳的其他維度的原始數值 |

**可用的 Liquid 過濾器：**

您可以使用 [Liquid 過濾器](https://liquidjs.com/filters/overview.html) 進行數值轉換：

```yaml
image:
  url: "https://example.com/${value.raw | downcase}.jpg"
```

### 目前限制

- **僅限維度**：圖片顯示目前僅支援維度與額外維度（不支援指標或表格計算）。
- **引用欄位**：使用 `${ row.table_name.column_name }` 範本時，被引用的欄位必須包含在您的結果表格中，圖片才能正常顯示。

## 在欄位名稱中使用特殊字元或大寫字母

如果您在欄位名稱中使用特殊字元或大寫字母，在探索數據時可能會遇到錯誤。例如，在 Postgres 的 `orders` 表中，如果欄位名稱為 `Status`（S 大寫），系統會拋出以下錯誤：

```bash
column orders.status does not exist
```

要修正此問題，我們可以在維度的 `sql` meta 標籤中為欄位名稱添加引號：

<Tabs>
  <Tab title="dbt v1.9 及更早版本">
    ```yaml
    - name: status
      meta:
        dimension:
          type: string
          sql: '"orders"."Status"' # 您也可以使用 '${TABLE}."Status"'
    ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
    ```yaml
    - name: status
      config:
        meta:
          dimension:
            type: string
            sql: '"orders"."Status"' # 您也可以使用 '${TABLE}."Status"'
    ```
  </Tab>
  <Tab title="Lightdash YAML">
    ```yaml
    dimensions:
      - name: status
        type: string
        sql: '"orders"."Status"' # 您也可以使用 '${TABLE}."Status"'
    ```
  </Tab>
</Tabs>

這會在生成的 SQL 查詢中正確引用 `Status` 欄位：

```sql
SELECT
  "orders".order_id AS "orders_order_id",
  "orders"."Status" AS "orders_status"
FROM "postgres"."jaffle"."orders" AS "orders"
```


## 額外維度 (Additional dimensions)

額外維度允許您基於 dbt 模型中的單個欄位定義多個維度。這在[套用不同的格式化](#添加不同的格式化)、[比較或結合欄位](#比較或結合欄位)、[解析 JSON 欄位](#解析-json-欄位)或建立基於某欄位的持久化分組/分桶時非常有用。

「普通」維度是在 dbt 的 .sql 檔案中建立並寫入數據倉庫的欄位。而額外維度不包含在 dbt .sql 檔案中，因此不會寫入數據倉庫。在 Lightdash 中使用時，它只是將維度定義加入到 SQL 查詢中（即在運行時建立）。

所有的[維度配置](#維度配置)都適用於額外維度。您也可以[在定義指標時使用額外維度](#在指標中使用額外維度)。

<Info>
  額外維度的名稱在模型中必須是唯一的。
</Info>

### 在指標中使用額外維度

若要定義基於額外維度的指標，您需要將它們添加到模型的 meta metrics 中，或者在欄位 meta 下定義指標時使用自定義 SQL。



<Tabs>
  <Tab title="dbt v1.9 及更早版本">
  ```yaml
  models:
    - name: users
      meta:
        metrics:
          highest_version_model_metric_example:
            type: max
            sql: ${version}
      columns:
        - name: metadata
          meta:
            dimension:
              hidden: true
            additional_dimensions:
              version:
                type: number
                sql: JSON_VALUE(${metadata}, '$.version')
            metrics:
              highest_version:
                type: max
                sql: ${version}
  ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
  ```yaml
  models:
    - name: users
      config:
        meta:
          metrics:
            highest_version_model_metric_example:
              type: max
              sql: ${version}
        columns:
          - name: metadata
            meta:
              dimension:
                hidden: true
              additional_dimensions:
                version:
                  type: number
                  sql: JSON_VALUE(${metadata}, '$.version')
              metrics:
                highest_version:
                  type: max
                  sql: ${version}
  ```
  </Tab>
  <Tab title="Lightdash YAML">
  ```yaml
  type: model
  name: users

  dimensions:
    - name: metadata
      hidden: true
    - name: version
      type: number
      sql: JSON_VALUE(${metadata}, '$.version')

  metrics:
    highest_version_model_metric_example:
      type: max
      sql: ${version}
    highest_version:
      type: max
      sql: ${version}
  ```
  </Tab>
</Tabs>

### 在額外維度中引用時間間隔

您可以在額外維度中引用特定維度的時間間隔。這對於建立基於特定時間粒度的自定義邏輯非常有用。

例如，您可以根據特定的時間間隔是否具有數值來建立一個分層 (Tier)：



<Tabs>
  <Tab title="dbt v1.9 及更早版本">
  ```yaml
    - name: session_start
      meta:
        dimension:
          type: timestamp
          time_intervals:
          - DAY_OF_WEEK_NAME
          - WEEK
          - MONTH
          - RAW
          - DAY
          - HOUR_OF_DAY_NUM
          - QUARTER
        additional_dimensions:
          extra_session_start_tier:
            type: string
            sql: |
              CASE
                WHEN ${session_start_month} IS NOT NULL THEN 'month'
                ELSE 'else' END
  ```
  </Tab>
  <Tab title="dbt v1.10+ 與 Fusion">
  ```yaml
    - name: session_start
      config:
        meta:
          dimension:
            type: timestamp
            time_intervals:
            - DAY_OF_WEEK_NAME
            - WEEK
            - MONTH
            - RAW
            - DAY
            - HOUR_OF_DAY_NUM
            - QUARTER
          additional_dimensions:
            extra_session_start_tier:
              type: string
              sql: |
                CASE
                  WHEN ${session_start_month} IS NOT NULL THEN 'month'
                  ELSE 'else' END
  ```
  </Tab>
  <Tab title="Lightdash YAML">
  ```yaml
  dimensions:
    - name: session_start
      type: timestamp
      time_intervals:
      - DAY_OF_WEEK_NAME
      - WEEK
      - MONTH
      - RAW
      - DAY
      - HOUR_OF_DAY_NUM
      - QUARTER
    - name: extra_session_start_tier
      type: string
      sql: |
        CASE
          WHEN ${session_start_month} IS NOT NULL THEN 'month'
          ELSE 'else' END
  ```
  </Tab>
</Tabs>


在此範例中，額外維度 `extra_session_start_tier` 引用了 `${session_start_month}`，即 `session_start` 維度的 `MONTH` 時間間隔版本。
