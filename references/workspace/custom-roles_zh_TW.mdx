## 概覽

Lightdash 提供兩種類型的角色：

- **系統角色**：預先定義的角色（如 Admin, Developer, Viewer），具備標準的權限集。欲了解這些角色的詳細能力，請參考我們的[預設系統角色權限矩陣](https://docs.lightdash.com/references/roles#roles-and-permissions)。
- **自定義角色**：由使用者定義的角色，您可以挑選特定的權限範圍以完全符合您的需求。

自定義角色是在組織層級建立的，但會在專案層級分配給使用者或群組，讓您可以按專案控制造訪權限。

<Info>
  **累加權限 (Additive Permissions)**：Lightdash 採用累計式的權限模型。如果使用者已經透過其組織層級角色獲得了某項權限，分配自定義角色將不會限制該造訪權限。自定義角色只能「增加」權限，不能移除現有的權限。
</Info>

## 建立自定義角色

### 造訪自定義角色設置

1. 導覽至 **Settings** → **General Settings** → **Custom Roles**。
2. 您將看到組織中現有的自定義角色列表。

### 從頭開始建立新角色

1. 點擊 **Create New Role (建立新角色)**。
2. 輸入 **Role Name (角色名稱)**（例如：「行銷分析師」、「財務檢視者」）。
3. 添加選填的 **Description (描述)** 來解釋角色的用途。
4. 選擇您想包含的特定 **scopes (權限範圍)**：
   - **View permissions (檢視權限)**：允許使用者查看內容（儀表板、圖表、空間）。
   - **Create permissions (建立權限)**：允許使用者建立新內容。
   - **Manage permissions (管理權限)**：允許使用者編輯、刪除或管理內容。
5. 點擊 **Save** 儲存角色。

![建立新角色](/images/references/workspace/create-new-role.png)

### 複製現有角色

如果您想建立一個與現有角色相似的角色：

1. 找到您想複製的角色（系統角色或自定義角色）。
2. 點擊角色旁的 **⋯** 選單。
3. 選擇 **Duplicate Role (複製角色)**。
4. 為新角色輸入名稱。
5. 根據需要修改權限範圍。
6. 點擊 **Save**。

這在您想以系統角色為基礎並進行微調時非常有用。

![複製角色](/images/references/workspace/duplicate-role.png)

## 分配自定義角色

自定義角色是在專案層級分配的，以提供精細的存取控制：

### 分配給使用者

1. 前往 **Project Settings** → **Access**。
2. 找到您想分配角色的使用者。
3. 從下拉選單中選擇自定義角色。
4. 該使用者現在將擁有此專案中該自定義角色所定義的權限。

### 分配給群組 (Groups)

1. 前往 **Project Settings** → **Access**。
2. 找到您想分配角色的群組。
3. 從下拉選單中選擇自定義角色。
4. 該群組的所有成員都將繼承此專案中的該自定義角色權限。

## 權限範圍 (Scope) 參考

### 管理 SQL 執行器 (SQL Runner) vs 管理自定義 SQL (Custom SQL)

這兩個權限範圍是獨立的，分別控制不同的功能。

**管理 SQL 執行器 (Manage SQL Runner)** 控制對 [SQL 執行器](/guides/developer/sql-runner_zh_TW) 的造訪權限，讓使用者能直接對數據倉庫編寫並執行即時 SQL 查詢。它還控制著建立[虛擬視圖](/guides/developer/virtual-views_zh_TW)以及從 SQL 執行器查詢[寫回 dbt 模型](/guides/developer/dbt-write-back_zh_TW)的能力。

**管理自定義 SQL (Manage Custom SQL)** 控制在探索 (Explore) 視圖中建立[自定義 SQL 維度](/guides/custom-fields_zh_TW)的能力。自定義 SQL 維度讓使用者能使用原始 SQL 向現有表格添加計算欄位。此權限**不**包含造訪 SQL 執行器的權限。

<Tip>
  如果您希望使用者僅查詢包含他人建立的自定義 SQL 維度的表格，他們不需要這兩項權限。任何具備探索視圖造訪權限的使用者都可以**使用**現有的自定義 SQL 維度 —— 這些權限範圍僅控制誰可以**建立**它們。
</Tip>

## 故障排除

### 使用者看不到預期的內容

- 驗證該自定義角色是否包含必要的檢視 (view) 權限範圍。
- 檢查角色是否已分配到內容所在的正確專案層級。
- 請記住，組織層級的權限可能會覆蓋自定義角色的限制。

### 角色變更未生效

- 使用者可能需要重新整理頁面才能使角色變更生效。
- 驗證角色是否已成功儲存並分配給正確的使用者/群組。

## 管理現有的自定義角色

### 編輯自定義角色

1. 前往 **Settings** → **General Settings** → **Custom Roles**。
2. 點擊您想編輯的角色。
3. 修改名稱、描述或權限範圍。
4. 點擊 **Save** —— 變更將套用到所有被分配此角色的使用者與群組。

### 刪除自定義角色

1. 首先，確保沒有使用者或群組被分配此角色。
2. 前往 **Settings** → **General Settings** → **Custom Roles**。
3. 點擊角色旁的**垃圾桶圖示**。
4. 確認刪除。

<Info>
  **刪除角色**：一旦刪除，自定義角色將無法恢復。您無法刪除目前正分配給使用者或群組的角色。
</Info>

自定義角色為管理 Lightdash 組織的造訪權限提供了強大的靈活性。透過精心設計符合團隊職責與工作流程的角色，您可以確保使用者獲得所需的精確權限，同時維持安全性與組織性。
