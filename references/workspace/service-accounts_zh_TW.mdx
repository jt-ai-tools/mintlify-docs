<Info>
  服務帳號需要 **企業版授權金鑰 (Enterprise License Key)**。
</Info>

## 概覽

服務帳號為自動化造訪 Lightdash API 提供機器對機器的身份驗證。與綁定個別使用者的個人存取權杖不同，服務帳號是為應用程式、CI/CD 管道與自動化工作流程設計的組織層級實體。

服務帳號讓您可以：

- 在不使用個人認證的情況下驗證 API 請求
- 授予特定的權限範圍（檢視、編輯或管理）
- 為權杖設定有效期限

## 啟用服務帳號（僅限自託管）

如果您是自託管 Lightdash，需要在配置中添加以下環境變數：

```bash
SERVICE_ACCOUNT_ENABLED=true
```

啟用後，重啟您的 Lightdash 執行體以使變更生效。

## 建立服務帳號

1. 導覽至 **General Settings** → **Service Accounts**
2. 點擊 **Add Service Account (新增服務帳號)**

   ![新建服務帳號](/images/references/workspace/new-service-account.jpg)
3. 輸入 **Description (描述)** 與選填的 **Expiry (到期時間)**
4. 選擇一個或多個要授予的 **Scopes (權限範圍)**
5. 點擊 **Create service account**
6. **複製** 生成的權杖。這是唯一一次顯示該權杖的機會。所有服務帳號權杖都會以 `ldsvc_` 為前綴。

## 權限範圍 (Scopes)

權限範圍具備層級性 —— 授予高階權限（如 `org:admin`）會自動包含所有低階權限（如 `org:edit`, `org:view`）。

| 權限範圍 | 描述 |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `org:view` | 對 Lightdash 內容的唯讀存取權。 |
| `org:edit` | 可建立/修改 Lightdash 內容（圖表與儀表板），**並包含**所有 `org:view` 權限。 |
| `org:admin` | 完整的組織層級控制權。包含部署 (deploy)、預覽 (preview)、下載/上傳、使用者管理，**並包含**所有 `org:edit` 與 `org:view` 權限。 |

## 使用服務帳號

### 配合 Lightdash CLI 使用

您可以使用服務帳號權杖來驗證 Lightdash CLI：

```bash
lightdash login https://your-lightdash-cloud --token ldsvc_your-token
```

### 配合 API 使用

服務帳號權杖使用 `Authorization` 標頭：

```bash
curl -H "Authorization: ldsvc_abc123xyz..." 
  "$LIGHTDASH_URL/api/v1/..."
```

欲了解完整的 API 文件，請參考 [API 參考指南](/api-reference/v1/introduction_zh_TW)。

## 範例使用場景

### CI/CD 自動化

在您的 CI/CD 管道中使用服務帳號自動將 dbt 變更部署到 Lightdash。詳情請參考[使用 GitHub Actions 部署](https://docs.lightdash.com/references/dbt-projects_zh_TW#1-自動部署：使用-github-action-將變更部署到-lightdash)。

### 執行體初始化

在設置新的自託管執行體時，配置 `LD_SETUP_SERVICE_ACCOUNT_TOKEN` 以透過服務帳號進行初始化。詳情請參閱[環境變數指南](https://docs.lightdash.com/self-host/customize-deployment/environment-variables_zh_TW#初始化執行體)。

### 自動化報告

使用服務帳號以程式化方式導出儀表板數據、排程報告，或透過 API 將 Lightdash 數據整合到其他系統中。
