<Info>
  **自定義視覺化是 Beta 版功能。**

  這意味著我們目前提供的文件和支援有限。我們未來也可能會更改這些選項的運作方式。
</Info>


### 已知限制

雖然這些由 Vega-lite 驅動的新圖表提供了更強的靈活性和更廣泛的視覺化選擇，但仍有一些重要的限制需要注意，特別是在互動功能和儀表板整合方面：

- **鑽取功能 (Drill-Down)**：自定義圖表中無法使用透過維度對指標進行分組的鑽取功能（例如「Drill by」）。
- **查看底層數據 (View Underlying Data)**：使用者無法點擊自定義圖表中的數據點來查看組成該數據點的底層記錄。
- **交叉過濾 (Cross-Filtering)**：自定義圖表目前不支援透過選取圖表元素來對儀表板其他磚塊進行互動式過濾（即交叉過濾）。


## 快速入門

建立自定義圖表的步驟如下：

<Steps>
  <Step title="收集數據">
    像平常一樣使用 Lightdash UI 選擇相關的維度和指標，獲取圖表所需的數據。
  </Step>
  <Step title="切換至自定義圖表">
    前往圖表配置選項，在下拉選單中，只要該功能已啟用，您應該會看到「Custom charts」的選項。
    ![](/images/references/chart-types/custom-charts-drop-down.png)
  </Step>
  <Step title="編寫配置">
    選擇一個範本來載入選定圖表的範例配置，或者直接抓取一個現成範例。
    <Tip> **提示：** LLM 非常擅長協助更新 Vega Lite 的配置代碼。 </Tip>
  </Step>
</Steps>

以下是一些我們在 Lightdash 中成功運行的圖表配置範本範例，以及各類圖表適用時機的建議。


## Vega 範本


### 長條圖 (Bar chart)

雖然您已經可以在 Lightdash 中使用內建預設來完成，但這是一個您可以快速上手的簡單範例。此圖表最適合搭配簡單的字串或日期維度，以及一個數值指標。


<Accordion title="帶有工具提示與樞紐的配置範例">
```json
{
  "$schema": "https://vega.github.io/schema/vega-lite/v5.json",
  "mark": "bar",
  "encoding": {
    "x": {
      "field": "orders_order_date_week",
      "type": "temporal" // 如果維度是字串或數字，請使用 quantitative
    },
    "y": {
      "field": "orders_total_order_amount",
      "type": "quantitative"
    },
    "color": { // 用於數據樞紐的選填屬性
      "field": "orders_status",
      "type": "nominal"
    },
    "tooltip": [ // 用於顯示工具提示的選填屬性
      {"field": "orders_order_date_week", "type": "temporal", "title": "訂單日期週"},
      {"field": "orders_total_order_amount", "type": "quantitative", "title": "總訂單金額"},
      {"field": "orders_status", "type": "nominal", "title": "狀態"}
    ]
  }
}
```
</Accordion>

它非常適合用於展示使用者同類群組 (Cohorts)、漏斗階段或隨時間變化的調查回應。任何有助於追蹤群組在各步驟間移動或切換的場景都適用。

### 熱圖 (Heatmaps)

熱圖非常適合視覺化橫跨兩個分類或時間維度的模式，讓人一眼就能識別趨勢、熱點或異常。它們非常適合分析基於時間的模式（如週天與小時的對應）、比較各類別的表現，或識別多維數據中的相關性。

此圖表最適合使用字串或日期維度，並以數值指標作為顏色深淺。

[在我們的示範網站查看此圖表的即時展示](https://demo.lightdash.com/projects/2014e038-ff4b-4761-ae6f-fbf551e7b468/saved/86a8e5d3-75e9-49ec-b65a-b3c6a4730217)。

在此範例中，熱圖使用訂單數據來顯示訂單週（X 軸）與一週中的天數（Y 軸）之間的模式，顏色強度代表訂單數量。下方的配置將輸出一個標準的熱圖：

![](/images/references/chart-types/custom-charts-heatmap.png)

<Accordion title="熱圖配置範例">

      {
        "field": "dbt_orders_order_date_week",

此圖表最適合使用字串或日期維度，並以一個數值指標作為 Y 軸，另一個數值指標作為氣泡大小。

[在我們的示範網站查看此圖表的即時展示](https://demo.lightdash.com/projects/7992ac17-dbb3-4c8e-8658-fcbc96eea949/saved/ec504908-e693-4b89-98d2-c06085a2f815)。

### 漏斗圖 (Funnel charts)

漏斗圖非常適合視覺化流程中數量逐步遞減的情況，例如銷售管道、轉化率或下單流程。每個階段都由一個長條表示，其寬度反映了對應的數值。

此圖表最適合使用分類或字串維度（如狀態或步驟名稱）以及數值指標（如訂單金額或數量）來衡量每個步驟。

漏斗圖的配置比其他圖表更複雜。我們建議直接在 Lightdash 中加載範本，它會自動映射您的數據欄位。

[在我們的示範網站查看此圖表的即時展示](https://demo.lightdash.com/projects/d496d901-a76d-4916-9eae-b81bc7337013/saved/b19b6a9f-b7a7-4795-bbd7-48f39b425c69)。

### 瀑布圖 (Waterfall charts)

瀑布圖展示了初始值如何受到一系列正向與負向變化的影響，讓人能輕易看出一連串增減所產生的累積效果。它們非常適合視覺化財務變動（營收分解、預算分析）、客戶生命週期指標，或任何需要追蹤各步驟對總額貢獻的場景。

此圖表最適合使用分類或時間維度作為 X 軸（如階段或時段），並以數值指標代表每個步驟的起始值與結束值。

[在我們的示範網站查看此圖表的即時展示](https://demo.lightdash.com/projects/d496d901-a76d-4916-9eae-b81bc7337013/saved/3706f242-e4c5-4bb7-9037-e750e18a06e4/)。

### 地圖 (Map charts)

此圖表最適合當您擁有 **經緯度欄位** 或者帶有 ID / 唯一名稱的 TopoJSON 查找表，以及用於調整點大小或顏色深淺的數值指標（如總銷售額或客戶數量）。
