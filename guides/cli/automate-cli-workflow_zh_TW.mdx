我們以 Github Actions 為例，因為它在我們的使用者中最受歡迎，但您可以使用任何能執行終端機指令的 CI/CD 工具。

我們包含了兩個最熱門自動化流程的模板：

1. [向 Pull Request 添加 Lightdash 預覽專案](#向-pull-request-添加預覽)
2. [將 dbt 變更部署到生產環境的 Lightdash 專案](#將變更部署到-lightdash)

這兩個工作流程都需要您向 GitHub 儲存庫添加一些秘密 (Secrets)，以便 CI/CD 工具能與 Lightdash 和您的數據倉庫進行驗證。我們首先來看看這部分。


## 向 Github Secrets 添加憑據

我們將向 GitHub Actions 添加一些秘密和配置，但您不希望這些資訊公開，因此最佳做法是將它們添加為 Github 上的 Secrets。

<Info>
  如果您已經有針對 Lightdash 的 GitHub Action，則可以使用為該 Action 建立的相同 Lightdash 秘密。
</Info>

在 Github 中，前往您的儲存庫，點擊 `Settings (設置)`，在左側邊欄的 `Security (安全)` 下點擊 `Secrets`。現在點擊 `New repository secret (新建儲存庫秘密)`。

<Frame>
  <img src="/images/guides/cli/github-secrets-fc35152c7ebb768974b385a0e2601dc5.png" />
</Frame>

我們需要添加 `LIGHTDASH_API_KEY` 作為秘密。

回到 Lightdash，透過前往 `Settings` > `Personal Access Tokens` 建立一個新的個人存取權杖。這就是您要在 `LIGHTDASH_API_KEY` 中填入的權杖。

<Frame style={{ marginBottom: '24px' }}>
  <img src="/images/guides/cli/personal-access-token-9ed8a8c2b16edc5ee745c4909d4093d3.png" />
</Frame>

`LIGHTDASH_PROJECT` 是您專案的 UUID。例如，如果您的網址看起來像 `https://app.lightdash.cloud/projects/3538ab33-dc90-aabb-bc00-e50bba3a5f69/tables`，那麼 `3538ab33-dc90-aabb-bc00-e50bba3a5f69` 就是您的 `LIGHTDASH_PROJECT`。

`LIGHTDASH_URL` 對於 Starter 客戶是 `https://eu1.lightdash.cloud` 或 `https://app.lightdash.cloud`，對於專用實例則是類似 `https://your_company.lightdash.cloud`。如果您是自託管，則應為您自己的自定義網域。

`DBT_PROFILES` 提示：

* 您可以從本地的 `profiles.yml` 檔案中複製大部分資訊。您可以透過在終端機中輸入 `cat ~/.dbt/profiles.yml` 來查看其中的內容。
* 如果您有獨立的 `prod` 和 `dev` 設定檔，您可能想為 GitHub Action 使用 `prod` 設定檔中的資訊。
* 如果您希望根據開啟 Pull Request 的使用者使用不同的連接設置（開發者設定檔），請[參考此指南](/guides/cli/how-to-use-lightdash-preview_zh_TW#如何在預覽專案中使用開發者憑據)。

從下方列表中尋找您的數據倉庫以獲取 profiles.yml 檔案模板。填寫此模板，這就是您的 `DBT_PROFILES` 秘密。

## 向 Pull Request 添加預覽

如果您已將 Lightdash 連接到 GitHub，您可以設置一個 `github action`，讓 Lightdash 在建立新的 `pull request` 時自動建立動態的 `preview` 專案，並在 `pull request` 關閉或合併時自動刪除該預覽專案。


### 建立預覽工作流程

前往您的儲存庫，點擊 `Actions` 選單，然後點擊 `Configure`。

![Github actions 頁面](/images/guides/cli/github-actions-f74f88aa1b1413a111be7dbf16d65d53.png)


如果您的儲存庫中已經有一些 GitHub Actions，請點擊 `New workflow`，然後選擇 `setup a workflow yourself`。

<Frame>
  <img src="/images/guides/cli/new-workflow-existing-actions-e321bd117d93bf139a01424d3771d0e6.jpg"/>
</Frame>

現在從我們的 [cli-actions](https://github.com/lightdash/cli-actions) 儲存庫中複製 [此 start-preview.yml 檔案](https://github.com/lightdash/cli-actions/blob/main/start-preview.yml)。

點擊 `Start commit` 進行儲存。

對 [此 close-preview.yml 檔案](https://github.com/lightdash/cli-actions/blob/main/close-preview.yml) 執行相同操作。


#### 使用開發者憑據

在開發 dbt 時，您通常使用與生產環境不同的一組憑據和數據集/schema。以下是兩種根據開啟 Pull Request 的使用者進行設置的方法。

<Info>
  如果您使用 dbt cloud IDE 建立提交和 Pull Request，則需要額外幾個步驟。我們需要在 GitHub Action 中添加一個步驟來獲取建立 Pull Request 的使用者。

  ```yaml
  - uses: actions/github-script@v6
    id: get_pr_creator
    with:
      script: |
        return (
          await github.rest.repos.listPullRequestsAssociatedWithCommit({
            commit_sha: context.sha,
            owner: context.repo.owner,
            repo: context.repo.repo,
          })
        ).data[0].user.login;
      result-encoding: string

  ```

  複製以下模板時，應將 `${{ github.actor }}` 替換為 `${{steps.get_pr_creator.outputs.result}}`。
</Info>


#### 使用 Profile 目標 (Targets)

更新您的 `DBT_PROFILES`，為每位開發者提供 1 個目標。目標名稱應為他們的 GitHub 使用者名稱。

```yaml
jaffle_shop:
  target: prod
  outputs:
    prod:
      type: bigquery
      method: oauth
      keyfile: keyfile.json
      project: jaffle-shop
      dataset: prod
    katie: # 範例開發者 1，應為 GitHub 使用者名稱
      type: bigquery
      method: oauth
      keyfile: keyfile.json
      project: jaffle-shop
      dataset: dbt_katie
    jose: # 範例開發者 2
      type: bigquery
      method: oauth
      keyfile: keyfile.json
      project: jaffle-shop
      dataset: dbt_jose
```

然後，更新您的 GitHub Action，使用使用者名稱作為 `lightdash start-preview` 指令的 `--target` 標籤。

```yaml
run: lightdash start-preview --project-dir "$PROJECT_DIR" --profiles-dir . --name ${GITHUB_REF##*/} --target ${{ github.actor }}
```

#### 使用 Github 環境 (Environments)

為每位開發者設置一個 [GitHub 環境](https://docs.github.com/en/actions/deployment/targeting-different-environments/using-environments-for-deployment)，其中秘密是專門針對他們的。環境名稱應為他們的 GitHub 使用者名稱。然後，更新您的 GitHub Action 以使用該使用者名稱作為環境。

```yaml
jobs:
  preview:
    runs-on: ubuntu-latest
    environment: ${{ github.actor }}
```


#### 使用 dbt cloud schema

如果您正在使用 [dbt cloud 的持續整合作業 (CI job)](https://docs.getdbt.com/docs/deploy/ci-jobs)，您可以為預覽專案使用由 dbt cloud 建立的 schema (`dbt_cloud_pr_<job_id>_<pr_id>`)。

首先，我們需要向您的 profile.yml 檔案添加一個環境變數，dbt 將使用它來連接到正確的 schema。

```yaml
schema: "{{ env_var('DBT_SCHEMA') }}"
```

如果您使用的是 BigQuery，應使用 `dataset` 而非 `schema`。

然後我們需要在 GitHub Action 中添加一個步驟來獲取 pull request ID。

```yaml
- uses: actions/github-script@v6
  id: pr_id
  with:
    script: |
      if (context.issue.number) {
        // 如果存在則回傳 issue 編號
        return context.issue.number;
      } else {
        // 否則從提交中回傳 issue 編號
        return (
          await github.rest.repos.listPullRequestsAssociatedWithCommit({
            commit_sha: context.sha,
            owner: context.repo.owner,
            repo: context.repo.repo,
          })
        ).data[0].number;
      }
    result-encoding: string
```

之後，我們需要向「Lightdash CLI start preview」步驟添加一個新的環境變數，即 dbt cloud 將使用的 schema。

請注意，在此範例中我們假設作業 ID 為 `1234`。您需要將其替換為實際的作業 ID。

```yaml
env:
  # ... 保留現有環境變數
  DBT_SCHEMA: 'dbt_cloud_pr_1234_${{steps.pr_id.outputs.result}}'
```

現在 dbt 在預覽環境中執行時將使用正確的 schema。


### 您完成了！

每當您建立新的 Pull Request 時，都會在您的組織中建立一個以您的分支名稱命名的 Lightdash 預覽專案。您會在 Pull Request 的對話中看到由 github-actions 機器人留言的此連結。每當您對該分支進行更改時，預覽環境都會更新。一旦您關閉或合併 Pull Request，預覽專案就會被刪除。

您可以在 `Github actions` 頁面查看日誌：

![Github actions 停止預覽](/images/guides/cli/github-actions-stop-3e0499fdef89b6723e6c17f80553cdaa.png)

## 將變更部署到 Lightdash

如果您已將 Lightdash 連接到 GitHub，可以使用 GitHub Action 在新變更合併到 main 分支時自動部署專案。這是使 Lightdash 與您對 dbt 的更改同步的最簡單方法。


### 建立部署工作流程

前往您的儲存庫，點擊 `Actions` 選單。

如果您沒有任何 GitHub Actions，只需點擊 `Configure`。

<Frame>
  <img src="/images/guides/cli/github-actions-f74f88aa1b1413a111be7dbf16d65d53.png"/>
</Frame>

如果您的儲存庫中已經有一些 GitHub Actions，請點擊 `New workflow`，然後選擇 `setup a workflow yourself`。

<Frame>
  <img src="/images/guides/cli/new-workflow-existing-actions-e321bd117d93bf139a01424d3771d0e6.jpg"/>
</Frame>

現在從我們的 [cli-actions](https://github.com/lightdash/cli-actions) 儲存庫中複製 [此 deploy.yml 檔案](https://github.com/lightdash/cli-actions/blob/main/deploy.yml)。

給它取一個好記的名字，如 `deploy-lightdash.yml`。

點擊 `Start commit` 提交到您的儲存庫。


### 您完成了！

每當您將變更合併到 `main` 分支時，它都會自動將您的新配置部署到您的 Lightdash 專案中。

您可以在 `Github actions` 頁面查看日誌。

<Frame>
  <img src="/images/guides/cli/github-run-e24152442d7002253f6fd6dc66a9a24e.png"/>
</Frame>
